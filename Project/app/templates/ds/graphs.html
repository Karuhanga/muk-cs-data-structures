Graphs

A graph is a way of representing relationships that exist between pairs of objects i.e. A graph is a set of vertices and edges which connect them.

Main Applications

Graphs have applications in modeling many domains, including mapping, transportation, computer networks, and electrical engineering.

Advantages

As functions and arrays, graphs have the following advantages; Simple and easy to understand Adapts easly to different kinds of graphs Efficient access Graphs can be constructed at run-time

Drawbacks

As functions and arrays, graphs have the following drawbacks; Cannot be extended to accomodate queries about the set of Vertices or the set of Edges. Depending upon the compiler that compiles the functions may not be very efficient. In fact the worst case time could be proportional to the number of vertices. The graph must be known statically at compile time. Requires that graph access be a Command rather than a computation. The domain of Vertices must be a type that can be used as an index into an array.
