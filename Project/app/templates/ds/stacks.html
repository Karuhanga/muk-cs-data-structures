Main Applications

The stack can be used to store a function's exception handlers, so when an exception occurs, the stack can automatically "unwind" back through the return addresses to locate a suitable handler. Everytime the user enters something, if you need to reverse, just pop it off the top of the undo stack and push it onto the redo stack. Besides the above feature, there are a bunch of other uses of stacks not limited to; The stack is also used to store a function's local arguments since these only need to remain in memory when the function is actually invoked (they will fall from scope as soon as the function returns). Callers can also pass arguments to functions by pushing them onto the stack immediately after the return address.

Advantages

Stacks provide an automatic means of backtracking because variables are popped from the stack in the reverse order they were pushed onto the stack. Thus you can always find your way back to a given state. Every thread in a process has its own built-in stack, making it possible for functions to know from where they were called and thus return control to their callers. The callers simply push the return address onto the stack before passing control to the callee. The return address stays on the stack until the callee is ready to return control to the caller. Thus functions can call functions (including recursive calls to themselves) without losing track of where each individual call came from. The stack essentially provides a "breadcrumb trail".

Drawbacks

We can use stack internally in every program, but the disadvantage of stacks is we can't resize, no searching, takes more memory since we create memory statically.
